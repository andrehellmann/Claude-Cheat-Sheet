Was Projekte können

  • Eigene System-Instruktionen (Ton, Rolle, Regeln)
  • Dateien dauerhaft im Kontext hinterlegen
  • Alle Chats teilen denselben Kontext
  • Teamweiter Zugriff (Team/Enterprise)
  • Konnektoren & Skills pro Projekt konfigurieren
Beispiel: Projekt „Kunde XY": Briefing + CI-Regeln + KPIs → alle im Kontext.

Wann Projekte nutzen?

  • Laufende Kundenprojekte mit mehreren Beteiligten
  • Disziplin-Workflows (z.B. „SEO-Team")
  • Wiederkehrende Berichte & Reportings
  • Interne Wissensdatenbanken
  • Onboarding neuer Mitarbeiter
→ Projekte ersetzen „Copy-Paste-Kontext". Setup-Anleitung: Tab 15

Projekt-Instruktionen

  • Rolle, Ton, Format, Sprache festlegen
  • Do's & Don'ts hinterlegen
  • Bei jedem Chat automatisch aktiv
Beispiel: „Du bist SEO-Experte bei netzstrategen. Antworte datengestützt."

Projekt-Dateien

  • PDFs, Docs, CSVs, Bilder hochladbar
  • Immer im Kontext — kein Upload nötig
  • Ideal: Briefings, Guidelines, Preislisten

Klarheitsfragen

  • Claude fragt bei Unklarheiten aktiv nach
  • Spart Iteration — lieber 1 Frage mehr
  • In Projekt-Instruktionen steuerbar
Beispiel: „Frag immer nach Zielgruppe und Format, bevor du einen Text erstellst."

What Projects Can Do

  • Custom system instructions (tone, role, rules)
  • Files permanently in context
  • All chats share the same context
  • Team-wide access (Team/Enterprise)
  • Connectors & skills configurable per project
Example: Project "Client XY": briefing + CI rules + KPIs → everyone works with the same context.

When to Use Projects?

  • Ongoing client projects with multiple contributors
  • Department workflows (e.g. "SEO team")
  • Recurring reports & reportings
  • Internal knowledge bases
  • Onboarding new team members
→ Projects replace "copy-paste context". Set up once, always available. Setup guide: Tab 15

Project Instructions

  • Define role, tone, format, language
  • Document do's & don'ts
  • Active in every chat within the project
Example: "You are an SEO expert at netzstrategen. Answer with data, concrete actions."

Project Files

  • PDFs, docs, CSVs, images uploadable
  • Always in context — no upload needed each time
  • Ideal: briefings, guidelines, price lists

Clarifying Questions

  • Claude actively asks when unclear
  • Saves iteration — better one extra question
  • Controllable in project instructions
Example: "Always ask about target audience and format before writing a text."

NETZ-BOT ist der auf netzstrategen zugeschnittene Claude-Assistent — konfiguriert mit internen Skills, CI-Regeln, Gedächtniseinträgen und Konnektoren.

Grundprinzipien

  • Kontext geben: Ziel, Zielgruppe, Format
  • Rolle definieren: „Du bist SEO-Experte…"
  • Schritt-für-Schritt denken lassen
  • Beispiele mitgeben (positiv & negativ)
  • Länge & Format explizit angeben
Statt: „Schreib einen Blogpost über KI."
Besser: „600-Wörter-Blogpost für Entscheider im Mittelstand über KI in Marketingteams. Ton: sachlich. Struktur: Intro, 3 Punkte, CTA."

Klarheitsfragen

  • „Welche Infos brauchst du noch von mir?"
  • „Frag mich zuerst nach Zielgruppe und Format."
  • In Projekt-Instruktionen als Pflicht hinterlegen

Für Kundenprojekte

  • Briefing als Anhang mitgeben
  • Immer Zielgruppe & KPIs nennen
  • Branche/Wettbewerb als Kontext angeben
  • Ton & Sprache spezifizieren
Beispiel: „Hier ist das Briefing für [Kunde]. SEO-Strategie für Q3. Zielgruppe: B2B-Entscheider, 35–55 Jahre."

Dateien & Uploads

  • PDFs, Word, Excel, Bilder, CSVs hochladbar
  • Claude liest und analysiert direkt
  • Mehrere Dateien gleichzeitig möglich
Beispiel: „Hier sind 3 Wettbewerber-Angebote als PDF. Vergleich Leistungen in einer Tabelle."

Core Principles

  • Provide context: goal, target audience, format
  • Define a role: "You are an SEO expert…"
  • Ask it to think step by step
  • Provide examples (positive & negative)
  • Specify length & format explicitly
Instead of: "Write a blog post about AI."
Better: "600-word blog post for SMB decision-makers about AI in marketing teams. Tone: factual. Structure: intro, 3 points, CTA."

Clarifying Questions

  • "What info do you still need from me?"
  • "Ask me about target audience and format first."
  • Set as mandatory in project instructions

For Client Projects

  • Attach the briefing document
  • Always mention target audience & KPIs
  • Provide industry/competitor context
  • Specify tone & language
Example: "Here's the briefing for [Client]. SEO strategy for Q3. Target: B2B decision-makers, 35–55."

Files & Uploads

  • PDFs, Word, Excel, images, CSVs uploadable
  • Claude reads and analyses directly
  • Multiple files simultaneously possible
Example: "Here are 3 competitor proposals as PDF. Compare services in a table."

What Are Scheduled Tasks?

In Claude Cowork, workflows can be executed automatically on a schedule — like a cron job, but configurable in natural language.

How to set up
  • Cowork → create task (prompt or plugin)
  • "Schedule" → set time, weekday, interval
  • Once, daily or weekly
→ Computer must be on, Cowork runs in background. Available from Pro plan.
→ New: Claude Code Routines run cloud-based — Mac doesn't need to be online. → Tab 02

WEEKLY
  • Monday: MITE data → Excel weekly report → Drive
  • Friday: Asana status → Gmail draft
MONTHLY
  • 1st of month: Campaign data → report → Drive

  • Create plugin "Monthly SEO Report"
  • Schedule plugin for 1st of each month
  • Claude executes fully: fetch data → analyse → report → Drive
